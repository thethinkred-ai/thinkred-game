# ThinkRed Economic Simulator - Презентация проекта

## Обзор проекта

**ThinkRed Economic Simulator** - это образовательная историческая экономическая стратегия, предназначенная для обучения марксистской политэкономии через практический геймплей с полной интеграцией в платформу Stepik.

### Ключевая концепция

- **Обучение через игру**: Изучение сложных экономических концепций через интерактивный геймплей
- **Исторический контекст**: Прохождение через 7 исторических эпох от феодализма до коммунизма
- **Практическое применение**: Управление предприятиями и принятие экономических решений
- **Интеграция с образованием**: Полная синхронизация с курсами Stepik

---

## Архитектура и технологии

### Технологический стек

#### Frontend
- **React 18** с TypeScript для типобезопасности
- **Vite** для быстрой разработки и сборки
- **TailwindCSS** для адаптивного дизайна
- **Recharts** для визуализации экономических данных

#### Backend
- **Node.js + Express** с TypeScript
- **PostgreSQL** для хранения данных
- **JWT** для аутентификации
- **WebSocket** для реального времени

#### Интеграция
- **Stepik OAuth 2.0** для аутентификации
- **RESTful API** для синхронизации данных
- **Phaser.js** (запланировано) для игровой визуализации

### Структура проекта

```
thinkred-game/
├── client/          # React frontend
├── server/          # Node.js backend  
├── shared/          # Общие типы и константы
├── game/            # Game logic (Phaser.js)
├── scripts/         # Скрипты сборки и развертывания
└── docs/            # Документация
```

## Игровые механики

### Предприятия

6 типов предприятий с уникальными характеристиками:

1. **Мануфактура** - Ручное производство, базовый уровень
2. **Фабрика** - Машинное производство, высокая производительность
3. **Шахта** - Добыча сырья для промышленности
4. **Ферма** - Сельскохозяйственное производство
5. **Магазин** - Торговля и реализация продукции
6. **Исследовательский центр** - Технологическое развитие

### Экономическая модель

#### Ключевые показатели
- **Прибыль** = Доходы - Издержки
- **Прибавочная стоимость** (марксистская концепция)
- **Норма прибыли** = Прибыль / Капитал
- **Концентрация капитала** - Уровень монополизации

#### Циклические процессы
- **Фазы цикла**: Экспансия → Бум → Кризис → Депрессия → Восстановление
- **Триггеры кризисов**: Перепроизводство, падение спроса, социальные конфликты
- **Последствия**: Безработица, падение прибылей, социальная нестабильность

### Исторические эпохи

| Эпоха | Период | Доступные предприятия | Особенности |
|-------|--------|---------------------|-------------|
| Феодализм | 1500-1750 | Мануфактуры, фермы | Ранние формы капитала |
| Ранний капитализм | 1750-1850 | + Магазины | Торговый капитал |
| Промышленная революция | 1850-1900 | + Фабрики, шахты | Машинное производство |
| Монополистический капитализм | 1900-1950 | + Исслед. центры | Тресты и монополии |
| Современный капитализм | 1950-2000 | Все типы | Глобализация |
| Переход к социализму | 2000-2050 | Все типы | Социальные преобразования |
| Коммунизм | 2050+ | Все типы | Коммунистическое общество |

## Интеграция со Stepik

### OAuth аутентификация
- Бесшовный вход через учетную запись Stepik
- Автоматическая синхронизация профиля
- Безопасная обработка токенов

### Прогресс-трекинг
- **Синхронизация достижений**: Игровые успехи → Прогресс в курсах
- **Разблокировка контента**: Доступ к новым урокам зависит от игрового прогресса
- **Анализ решений**: Отправка экономических решений для педагогического анализа

### Образовательная интеграция
- **Адаптивное обучение**: Сложность контента зависит от уровня игрока
- **Практические задания**: Игровые сценарии привязаны к теоретическим урокам
- **Оценка понимания**: Проверка знаний через практические решения

## Метрики успеха

### Образовательные метрики
- **Вовлеченность студентов**: Время в игре, количество решений
- **Понимание концепций**: Результаты тестов до/после игры
- **Прогресс в курсах**: Корреляция с игровыми достижениями
- **Обратная связь**: Оценка полезности игры

### Технические метрики
- **Производительность**: Время загрузки, отзывчивость
- **Доступность**: Uptime, время отклика API
- **Масштабируемость**: Количество одновременных пользователей
- **Надежность**: Количество ошибок, сбоев

### Бизнес-метрики
- **Количество пользователей**: Рост базы пользователей
- **Удержание**: Процент возвращающихся пользователей
- **Завершенность**: Процент проходящих игровые сценарии
- **Интеграция**: Успешность синхронизации со Stepik
